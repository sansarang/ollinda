# GEO — AI 검색(ChatGPT·Perplexity 등)이 내 가게/상품을 인용하게 (B블록)

2026 트렌드: 가게·상품 탐색이 네이버 검색만이 아니라 **AI 검색(생성형 답변)**으로 급증.
AI는 답을 만들 때 **구조가 명확한 글**(정의문·Q&A·요약·일관 표기)을 인용하기 쉽다.
올린다는 블로그 글을 그 구조로 쓰게 해 "AI 답변에 내 가게가 인용될 확률"을 높인다.

## 원리 — 무엇을 넣나 (`app/seo.py`)

| 요소 | 매장형 (NAP) | 셀러형 (SPU) |
|---|---|---|
| 정의문 | 첫 문단에 "○○는 부산 동구의 썬팅 전문점이다" | "△△ 폴딩박스는 60L 캠핑 수납박스다" |
| 한눈 요약 | `## 한눈 요약` 핵심 3줄 목록 | 동일 |
| Q&A | 실제 검색 질문형 FAQ ("부산 동구 썬팅 얼마나 걸리나요?") | 비교 Q&A 포함 ("○○ vs 유사제품 차이?") |
| 솔직함 | — | `## 솔직 장단점` (아쉬운 점 1개 — 입력 근거만, 날조 금지) |
| 표기 일관 | 상호·주소·전화(NAP) 본문 전체 동일 표기 | 상품명·스토어·구매링크(SPU) 동일 표기 |

- `geo_directive(biz_type, name, industry, region, brand, questions)` — 블로그 생성 프롬프트에 주입(`text_claude.BlogDraftGenerator`). 매장/셀러 분기.
- `geo_questions(industry, region, pain_points)` — 업종 프로필 pain_points → "AI가 받을 질문" 3개 자동 생성.
- 필수 섹션에 `## 한눈 요약` 추가(기존 FAQ·표에 더해).

## 점수 — AI검색 준비 점수 (`seo.geo_audit`, LLM 0콜)

구조 요소 기계 채점: 정의문 · 한눈 요약 · Q&A · 표기 일관(+셀러는 솔직 장단점). 항목 균등 배점 0~100.
`ingest` 생성 직후 blog payload에 `geo_audit` 저장 → 결과 화면 헤더에 **"상위노출 N점 + AI검색 준비 M점" 병기**
(보라 뱃지, 툴팁: "인용을 보장하진 않아요").

## 정직 원칙 · 한계 (셀링에도 이 이상 말하지 않는다)

1. **인용 보장 불가** — GEO는 "인용되기 유리한 구조"일 뿐, AI가 실제로 인용할지는 모델·시점에 따라 다르다.
   랜딩 문구도 "인용되기 유리한 구조로 씁니다"까지만(보장 표현 금지).
2. **색인 시간** — AI 검색이 새 글을 읽으려면 웹 색인에 잡혀야 한다. 발행 후 수일~수주 걸릴 수 있다.
3. **측정 한계** — "AI가 내 가게를 인용했는지" 자동 측정은 현재 불가(각 AI 서비스에 질의 API로 물어보는 방식은
   비용·약관 문제). 점수는 '구조 준비도'지 '인용 실적'이 아니다.
4. **날조 금지** — 솔직 장단점·비교 Q&A는 입력 정보 근거만. 타사 비방·비교 우위 날조 금지(FACTS_RULE 상속).

## 셀링 (landing.py — 문구 1줄만 추가)

히어로: "네이버만? **ChatGPT·AI 검색에도 내 가게가** — 인용되기 유리한 구조로 씁니다"

## 검증

- 단위: `geo_audit` 매장 만점/구조없음 0점/셀러 만점 케이스, 프롬프트 주입 여부, 점수 병기 렌더 (B1·B2 커밋에 포함).
- 실생성: 프로덕션 배포 후 샘플 글에서 정의문·한눈 요약·Q&A·(셀러)솔직 장단점 포함 확인 + 결과 화면 병기 뱃지 확인 (B3).
