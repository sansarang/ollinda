# BLOG_TEMPLATE — 업종별 블로그 템플릿 + 네이버 컴포넌트 가이드

네이버 스마트에디터3.0은 텍스트·사진·지도를 **컴포넌트**로 다룬다.
핵심 전제: **지도·장소·연락처는 텍스트로 박지 말고 네이버 자체 '장소' 컴포넌트로** 넣어야
플레이스 연결·지역SEO에 유리하다. 이미지 붙여넣기는 불안정하므로 **파일 업로드**가 안전(기존 [사진N] 방식 유지). 모바일 우선.

## PHASE 1 — 매장 고정정보 (한 번 입력 → 모든 글 재사용)

- `tenants`: 기존 phone/address/hours/map_url/buy_url/search_kw 재사용 + **`parking`(주차) 신규**.
- 저장: `db.update_store_info()` (parking 컬럼 없는 구DB도 graceful).
- UI: `/me` 홈 하단 "🏪 매장 정보" 카드(`_store_info_card`) — 입력 2개 미만이면 자동 펼침,
  셀러(seller/hybrid)는 구매링크·검색어 필드 추가. `POST /me/store-info`.
- 온보딩 유도: 온보딩 저장 시 매장형인데 주소·전화가 비면 "매장 정보까지 채우면 모든 글에 자동 삽입" 안내.

## PHASE 2 — 업종별 블로그 템플릿 (`app/services/blogtpl.py`)

기존 매장형/셀러형 자동분기(`tenant.biz_type` → strategies)를 재사용해 시퀀스를 고정한다.
업종 톤은 기존 `industries.py` 페르소나/톤 프롬프트가 그대로 결합된다(수정 없음).

```
매장형: 제목 → 도입(PAS) → [사진1] → 본문(1인칭 경험, ##소제목) → [사진N]
        → Q&A 3쌍 → [고정정보블록: 자동삽입] → 해시태그
셀러형: 제목 → 도입(구매 전 고민) → [사진1](상품) → 후기 본문(장점+아쉬움 솔직)
        → Q&A 3쌍 → [구매블록: 자동삽입] → 해시태그
```

- `sequence_directive(biz_type)` → BlogDraftGenerator 프롬프트에 주입(`_tpl_sequence`).
  LLM에게 "고정정보·링크는 시스템이 자동 삽입하니 본문에 쓰지 마라(중복 금지)"를 지시.
- **고정정보 블록** `fixed_info_block(tenant)`: 주소·전화·영업시간·주차는 텍스트(복사 가능),
  지도는 URL 대신 **`[여기 네이버 지도 넣기]` 마커**(`MAP_MARKER`)만 삽입 → 발행 화면 가이드로 연결.
  마무리에 "네이버에서 '{상호}' 검색 → 플레이스 저장·리뷰·예약" 행동 유도(플레이스 신호).
- 셀러는 기존 `strategies.buy_block` 재사용(`seller_buy_block`), hybrid는 둘 다(`closing_block`).
- payload에 `fixed_info_block` 저장 → 발행 화면(PHASE 3)이 참조.

## PHASE 3 — 네이버 컴포넌트 삽입 가이드 (`_naver_component_guide`, 모바일 우선)

발행 화면(`/kit/{id}/naver`)의 고정정보 위치에 표시:

1. **3스텝 진행 표시**: ① 네이버 글쓰기에서 '장소' 버튼 → ② 아래 상호 붙여넣기 → ③ 내 가게 선택.
2. **장소 검색용 "상호명+지역" 큰 복사 버튼**(초록, 전폭 터치 버튼) — 누르면 "✅ 복사됨" 피드백.
3. 플레이스 URL 등록돼 있으면 "내 플레이스 열어 확인" 링크 제공.
4. **연락처**: 텍스트로 충분 — 네이버가 전화번호를 자동 링크(tel:)하므로 별도 마크업 불필요를 안내.
5. **영업시간·주차**: 개별 복사 버튼(접이식).

모바일 배려: 전폭 큰 터치 버튼, 요소별 개별 복사, 스텝 도트 진행 표시.

## PHASE 4 — 임시저장/이어쓰기 워크플로우 (`_workflow_guide`)

네이버 블로그는 **PC↔모바일앱 임시저장이 동기화**된다. 발행 화면 상단에 상황별 탭:

| 상황 | 추천 흐름 |
|---|---|
| 💻+📱 둘 다 (추천) | PC에서 제목·본문 붙여넣기 → **임시저장** → 모바일앱 '임시저장 글 이어쓰기' → 사진·장소 컴포넌트 추가 → 발행 |
| 💻 PC만 | ZIP으로 사진 저장 → 붙여넣기+업로드 → 장소 컴포넌트 → 발행 |
| 📱 모바일만 | 요소별 복사 → 앱에 붙여넣기 → 사진은 저장 후 업로드(붙여넣기 불안정) → 장소 → 발행 |

각 흐름 마지막은 "발행함 ✓"(발행 확인, BLOG_TRACKING.md)으로 연결 → 순위 추적 루프 진입.

## 정직성

- 고정정보는 **사장님이 직접 입력한 값만** 삽입 — 빈 항목은 블록에서 생략(지어내지 않음).
- 지도 마커는 실제 삽입을 대신하지 않음을 명시("사용자가 직접 장소 컴포넌트로 삽입") — 자동 발행·에디터 조작 없음.
- 템플릿 지시문도 기존 FACTS_RULE(없는 가격·스펙 금지)·키워드 밀도 가드와 결합되어 동작.

## 관련 파일

- `app/services/blogtpl.py` — 시퀀스·고정정보·구매 블록
- `app/generators/text_claude.py` — `_tpl_sequence` 주입 + 고정정보 자동삽입(기존 fallback 블록 대체)
- `app/main.py` — `_store_info_card`/`POST /me/store-info`(P1), `_naver_component_guide`(P3), `_workflow_guide`(P4)
- `app/db.py` — `parking` 컬럼 + `update_store_info`
