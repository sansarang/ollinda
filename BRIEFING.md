# BRIEFING — 매일 아침 브리핑 (AI 사장님 파트너)

수동(사장님이 들어와야 일어남) → **능동**(올린다가 매일 아침 먼저 "오늘 상황 + 할 일 딱 하나").
심리 설계: ① 할 일 딱 하나(5개 주면 0개 함) ② 손실·경쟁 프레이밍은 해결책과 함께
③ "나머지는 제가 할게요" 파트너 톤. 하루 루프(아침 브리핑→실행→저녁 피드백)로 습관·리텐션.

## 브리핑 선정 로직 (`app/services/briefing.py`)

기존 재료(순위 추적·경쟁사 스냅샷·발행 이력·진단+실검색량)를 신호로 점수화 → **최고 1개만** 선정:

| 신호 | 점수 | 예시 브리핑 |
|---|---|---|
| 경쟁사 역전·추격 (`competitor_snapshots`) | 90 | "'부산 국밥'에서 부산왕국밥이 위에 있어요(1위 vs 내 가게 5위)" → 추격 글 1편 |
| 내 순위 하락 (`rank_snapshots` 실측 비교) | 80+낙폭 | "'부산 국밥' 2위→5위로 밀렸어요" → 새 글 1편 |
| 발행 공백 ≥3일 (`publish_activity`) | 70+일수 | "6일째 새 발행이 없어요" → 캘린더 제안 주제 1편 |
| 미노출 고검색량 키워드 (진단+searchad 실측) | 60(+량) | "월 1,200회 검색되는데 안 보여요" → 선점 글 1편 |
| 순위 상승 (굳히기) | 50+상승폭 | "오르는 중!" → 같은 키워드 1편 더 |
| 신호 없음 | 10 | **정직 폴백**: "특별한 변화 없어요 — 꾸준함이 곧 순위" |

브리핑 구조: `{headline(상황 한 줄), task(할 일 하나 — "사진 3장만"), reason(손실/경쟁 프레이밍+해결),
action_href(원클릭), partner_note("나머지는 제가")}`. tenant×날짜 1건 캐시(`daily_briefings` PK).

※ searchad "주간 급증"은 이력 데이터가 없어 실측 불가 → 지어내지 않고 '고검색량 미노출 기회'로 대체
(급증 감지는 volume 스냅샷 축적 후 후속 과제).

## 발송 스케줄 (`scheduler.py`)

| 잡 | 주기(KST) | 동작 |
|---|---|---|
| `morning_briefing` | 매시 정각 05~12시 | `send_morning(현재 KST시)` — `tenant.briefing_hour` 일치 가게만 |
| `evening_feedback` | 20:00 | `send_evening()` — **오늘 콘텐츠 만든 가게만**(안 만든 날 조용 = 비스팸) |

- 채널: 앱내 알림(notices) + 이메일(SMTP 설정 시, 게스트 주소 제외) + **카카오 알림톡 스텁**(`_send_kakao_stub` — 비즈메시지 연동 지점).
- 중복 방지: `daily_briefings.sent / evening_sent` 락(1일 1회), 스케줄러 1인스턴스 전제.
- 설정: `/me/briefing-pref` — 시각(05~12시)·on/off. 기본 08시 on. 수동 테스트: `POST /admin/briefing/send-now`.

## 원클릭 흐름 (PHASE 3)

브리핑 action → `/me?target_kw={키워드}&angle={앵글}&from=briefing` — **기존 스마트입력·생성 파이프라인 그대로**
(타겟 키워드·앵글 자동 주입). `from=briefing`이면 업로드 폼 배너가 파트너 톤으로:
"오늘 브리핑에서 제안드린 'X' — 사진 3장만 올려주세요. 글·영상·발행 준비는 제가 할게요" + **오늘은 패스** 버튼
(`POST /api/briefing/pass` → 카드가 "푹 쉬세요" 상태로, 내일 재개).

## 저녁 성과 피드백 (PHASE 4)

전부 실측(`db.today_feedback_stats`): 오늘 추적링크 클릭 수(link_clicks) · 오늘 스냅샷의 순위 변화(어제 대비) ·
오늘 생성 건수. **데이터 없으면** "내일부터 순위 변화 추적해서 알려드릴게요 — 네이버 반영엔 하루 이틀" 정직 안내.
하락이 잡히면 "내일 브리핑에서 대응책 드릴게요"로 다음 날 루프 연결.

## 정직성 가드

1. 모든 신호는 실측 테이블(rank/competitor/publish/link_clicks)만 — 임의 수치·가짜 급증·가짜 조회수 없음.
2. 신호 없는 날은 "변화 없음" 브리핑(지어내지 않음) — 랜딩 셀링 문구에도 이 원칙 명시.
3. 손실·경쟁 프레이밍은 반드시 해결책(오늘 할 일)과 함께. "무조건 1위" 금지.
4. 저녁 피드백은 실행한 날만(허위 성과·스팸 금지).

## 플랜 게이팅 결정: 전 플랜 제공

리텐션 목적(매일 들어올 이유 = 구독 유지·무료→유료 전환 동력)이라 **넓게 개방**.
브리핑이 유도하는 '생성'은 기존 무료 2회/플랜 한도가 그대로 게이트 역할 — 비용 통제는 생성 단에서.
추후 프리미엄 차별화는 카카오 알림톡(발송비 실비) 채널을 프로 전용으로 여는 방식 권장.

## 대시보드·랜딩 (PHASE 5)

- `/me` 상단 "오늘 아침 브리핑" 카드(연보라 강조): 헤드라인·할 일·이유·파트너 노트 + [사진 보내고 시작하기]·[오늘은 패스] + 접이식 설정(시각·on/off).
- 랜딩 `_briefing_sell` 섹션(연보라 밴드): 아침 브리핑→사진 3장→저녁 피드백 3카드 + 정직성 각주.
