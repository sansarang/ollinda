# DIFFERENTIATION — S(셀러)·A(휴먼터치)·B(GEO)·C(브리핑) 통합 (2026-07)

매장 중심이던 올린다를 **매장/셀러 양축**으로 확장하고, 2026 차별화 3종(휴먼터치·GEO·능동 브리핑)을
그 양축 위에 얹었다. 순서는 S 먼저(셀러 분기 토대) → A·B·C가 처음부터 양쪽으로 동작.

## 1. 기능 요약

| 블록 | 무엇 | 핵심 파일 |
|---|---|---|

## 2. 매장/셀러 매핑표

| 축 | 매장 (local/hybrid) | 셀러 (seller) |

| 진단 위젯 | 지역·업종·상호 입력 | 토글 시 상품 키워드·스토어명(지역 숨김), "40위 밖" 라벨 |

| 블로그 | 방문 유도, 고정정보(NAP) 자동 삽입 | 내돈내산 후기 톤, 구매 블록(SPU), + 마켓 상품명·상세·요약본·리뷰 키트 |

| 브리핑 신호 | 경쟁사 역전 90 > 순위 하락 80 > 발행 공백 70 > 미노출 키워드 60 > 상승 50 | 쇼핑 추격(1위 상품·가격 실측) 85 > 순위 변화(shop) > 발행 공백 > 쇼핑 미노출 60 |
| 발행 캘린더 | 지역+업종 주제, 후기/방법/가격 앵글 | search_kw/브랜드+상품 주제, 내돈내산/사용법·비교/가성비 앵글 (기존 C2) |

## 3. 정직성 가드 (전 블록 공통)

- 순위: 스캔 범위 밖은 임의 숫자 금지 — 매장 "미노출", 셀러 "40위 밖". 조회 불가는 '추정' 라벨.
- 브리핑: 실측 테이블만(rank/competitor/publish/shop.json 현재값). 신호 없으면 "변화 없어요" 폴백 — 억지 브리핑 금지.
- 셀러 가격 정보: shop.json **현재값**만 표기("지금 1위는 ○○몰 12,900원") — 이력 없는 '추세' 날조 금지.
- 리뷰 키트: 결정적 템플릿(LLM 미사용) — 대가성(사은품·포인트 조건) 문구 원천 배제 + 규정 경고 문구 노출.
- GEO: "인용되기 유리한 구조"까지만 — 인용 보장·측정 주장 금지(GEO.md 한계 명시).
- 휴먼터치: 경험담 없으면 억지 경험 배치 지시를 넣지 않음. 하이라이트도 "그대로 들어감/표현에 녹임" 구분 표기.
- 크롤링 없음 — 네이버 공식 API(local/shop/blog.json, searchad)만.

## 4. 스텁·한계

- **카카오 알림톡**: `briefing._send_kakao_stub` — 템플릿 승인 후 연결(기존과 동일).
- **쇼핑 API 한계**: shop.json은 **리뷰 수·평점 미제공** → "리뷰수 동향" 신호는 구현하지 않음(날조 방지).
  가격도 스냅샷 이력이 없어 '추세'가 아닌 '현재값'만. 이력 축적 후 추세 신호는 후속 과제.
  display 상한 100 — 40위 스캔은 호출 1회로 유지(쿼터 절약).
- **검색량 급증 감지**: volume 일일 스냅샷 미축적 — '고검색량 미노출'로 대체(BRIEFING.md와 동일).

## 6. 검증 기록 (2026-07-12 프로덕션 실측)

- 셀러 진단 e2e: `mode=seller` → 실검색량 월 55,200회 미노출 손실 프레이밍 + "40위 밖" 라벨.
- 휴먼터치 실생성 3종(빵집·썬팅·폴딩박스): 클리셰 0/1/0건, 경험담 3종 모두 도입부 배치, 문단 변동계수 0.18~0.32, "사장님 이야기" 배지 렌더.
- GEO 실생성: 매장·셀러 모두 한눈 요약·Q&A·(셀러)솔직 장단점 포함, "상위노출 100점 + AI검색 준비 60점" 병기 확인.
- 브리핑 샘플(send-test, 캐시 없는 신규 계정): 셀러 → `shop_missing` "'캠핑용품점' 월 3,920회… 상위 40위 안에 내 상품이 안 보여요. 지금 1위는 네이버(13,630원)" / 매장 → `missing_kw` "'부산 썬팅' 월 720회… 가게가 아직 안 보여요" — **양축 신호 분리 확인**.
- 발견·수정한 실버그: 셀러 타겟 키워드에 intake 라벨("사장님 제공 실제") 유출 → 제목 오염 (`c77b6ee`).
- 개선 여지(후속): 셀러 롱테일 연관어에 매장성 키워드('캠핑용품점')가 섞일 수 있음 — searchad `_relevant` 필터 정교화 과제.
