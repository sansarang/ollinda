# VIDEO_ENGINE — 2026 플랫폼 신호 기준 영상 생성

숏폼 도달을 좌우하는 2026 알고리즘 신호와 각 PHASE의 대응.
파이프라인: `app/generators/video.py` (씬 분할 → 씬별 TTS 싱크 → xfade 합성 → ASS 자막 → BGM mux).

## 플랫폼 신호 ↔ PHASE 매핑

| 2026 신호 | 근거 | 구현 (PHASE) |
|---|---|---|
| **첫 3초 완주율** (도달 5~10배) | 오프닝 훅이 최우선 | P1: 실사진 배경 훅(`_hook_photo_png`) + 손실회피 카피 4안 생성→`_pick_hook` 자동선택(손실회피어·숫자·키워드·8~16자 가점) + **첫 프레임 즉시 노출**(fade_in 제거, 인트로 없음) |
| **저장·공유 > 좋아요** (3~5배 가중치) | 정보/튜토리얼 포맷이 저장 높음 | P5: `SAVE_SHARE_RULE`(단계형 'OO하는 법 3단계' 우대), 아웃트로 "🔖 저장해두고" CTA, 캡션/설명 플랫폼별 CTA 자동삽입(`save_share_line`, 중복 방지) |
| **반복재생 유도** | 빠른 텍스트·빽빽한 정보·루프 | P2: `SUBTITLE_DENSITY`(씬당 구체 정보 1개), P4: 끝 페이드 제거 + 아웃트로=훅과 같은 실사진 배경(끝→처음 연결) |
| **워터마크 = 노출 감소** | 교차게시 페널티 | P4: 로고 워터마크 오버레이 **제거**(진행바는 유지 — 워터마크 아님), 품질 게이트가 `no_watermark` 확인 |
| **오리지널·주제 일관성** | 실사진 우대 | P1: 그라데이션 카드 대신 사장님 실사진 훅/아웃트로, 기존 켄번스·색보정 유지 |

## PHASE별 상세

- **P1 훅**: 프롬프트 `[훅후보]` 4안(훅 공식 4종 각각) → `_pick_hook` 점수 선택 → payload `hook_candidates` 보존. 실사진 cover-crop + 상하 그라데이션(중앙 사진 보존) + 상단 1/3 대형 텍스트(외곽선 6방향).
- **P2 자막**: `tts.synthesize_timed` — ElevenLabs `/with-timestamps` character 타임스탬프 → 단어 병합(`_chars_to_words`) → `_build_ass`가 실측 단어 타이밍으로 카라오케(`\k`) 생성, 실패/무키 시 글자수 근사 폴백. 폰트 78·외곽선 7·그림자 4·MarginV 380(하단 UI 안전영역 밖). 키워드는 테마색+굵기+106% 확대.
- **P3 오디오**: 목소리 `loudnorm I=-14 LUFS`(소셜 표준), BGM 0.30 + `sidechaincompress`(threshold 0.03 → 무음 구간 펌핑 방지) — 기존 유지. **업종별 무드**: `bgm.mood_for(업종)` → calm(카페·뷰티·의료)/upbeat(음식·리테일)/trust(자동차·법률·기술), `bgm/{mood}/` 하위폴더 → 파일명 키워드 → 전체 랜덤 순. BGM 소스는 상업 라이선스 확보 파일만 assets/bgm에 (운영자 관리).
- **P4 규격·루프·화질**:
  - 씬 전환 = **xfade 크로스페이드**(`_concat_xfade`): 각 클립에 tail 0.25s를 렌더해 전환이 tail을 소모 → 총 길이 = 오디오 길이(자막·음성 싱크 보존). 실패 시 tail 트림 후 concat 폴백.
  - 최종 화질 패스 = `_post_overlay` **veryfast -crf 20**(중간 클립은 ultrafast), mux에 `+faststart`.
  - 규격/길이는 아래 표 + 품질 게이트로 검증.
- **P6 안정성**: `_run_ff`(모든 ffmpeg 호출 stderr 로깅 — 기존 소실 문제 해결), 씬 탈락 카운트(note + 게이트), `RENDER_SEM`(동시 렌더 2, env `SHOPCAST_RENDER_CONCURRENCY`), 레거시 폴백도 drawtext 자막 유지(기존). `_quality_gate` → payload `quality_gate`.

## 규격·길이표

| 산출물 | 해상도 | 용도 | 길이 목표 |
|---|---|---|---|
| 마스터 (video_path) | 1080×1920 (9:16) | 유튜브 쇼츠·인스타 릴스 | 쇼츠 30~45초(완주율), 릴스는 15~30초 구간이 유리 — 프롬프트가 30~45초 목표로 기획, 씬 수(≤6)로 상한 제어 |
| square 변형 | 1080×1080 (1:1) | 인스타 피드 | 마스터와 동일(블러 배경 패딩) |
| feed45 변형 | 1080×1350 (4:5) | 인스타 피드 | 〃 |

품질 게이트 체크: `spec_9x16`(정확 규격) · `duration_ok`(8~62s) · `has_audio`(무음 TTS는 경고만) ·
`hook_first_frame` · `subtitles_burned` · `no_watermark` · `no_dropped_scenes`.

## 정직성

- 훅·내레이션도 기존 `FACTS_RULE` 적용 — 없는 가격·스펙·효능 금지, 과장어(무조건/100%/1위) 금지.
- 저장·공유 유도는 허용하되 **좋아요 구걸 금지**, 가짜 긴급성(오늘만/마감임박 날조) 금지.
- AI 보충 이미지는 사진 부족 시 최대 2장(기존 정책 유지) — 실사진이 항상 우선(오리지널 신호).

## 검증

로컬 ffmpeg로 실제 렌더 통합 테스트 통과: 실사진 훅 + 씬 3개 + xfade + 진행바, 1080×1920,
길이 = sum(씬 길이) 오차 0.5s 미만, 품질 게이트 전 항목 pass.
