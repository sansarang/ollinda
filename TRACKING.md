# TRACKING — 콘텐츠별 클릭·유입 실측

사장님이 "내 글이 몇 명을 데려왔는지" 숫자로 보게 하는 기능. 효과 실감 = 리텐션 핵심이며,
아침·저녁 브리핑의 실데이터 재료다.

## 측정 가능 / 불가 (정직 원칙)

| 항목 | 측정 | 왜 |
|---|---|---|
| 올린다 추적링크(/r/{code}) 클릭 | ✅ 행 단위 실측 | 우리 서버를 경유 — 시각·채널·콘텐츠·리퍼러·UA 기록 |
| 매장 QR 스캔 유입 | ✅ (src=qr) | QR도 추적링크 경유 |
| 네이버 블로그 조회수 | ❌ | 공식 API 없음 — **있는 척 표기 금지** |
| 인스타 노출·도달 | ❌ | 동일 |
| 클릭 후 실제 구매/방문 완료 | ❌ | 외부 사이트 이후는 추적 불가 |

UI 표기 원칙: "조회수"라 부르지 않는다. "링크 클릭(추적링크 경유 방문)"으로 표기하고
"실제 조회수는 네이버/인스타 앱 통계에서 확인하세요"를 병기한다. 뱃지 툴팁에도 명시.

## 추적링크 구조 (P1)

- URL: `/r/{code}?src={channel}&content={piece_id 앞8자}`
  - `code`: 가게 대표 목적지(매장=플레이스/지도, 셀러=스토어) 단축링크 — 가게당 재사용.
  - `src`: naver_blog / instagram / marketplace / qr … (utm_source는 하위호환)
  - `content`: 콘텐츠(피스) 식별 — 링크 남발 없이 콘텐츠 단위 어트리뷰션.
- 기록: `link_clicks(code, ts, referrer, ua, utm_source, content_id, channel)` 행 단위(카운터 아님).
- 본문 자동 포함(`tracklinks.inject`, polish 뒤 실행 — 리라이트로 유실 방지):
  - 셀러: 본문 속 원본 구매링크(buy_url)를 콘텐츠별 추적 URL로 치환.
  - 매장: map_url 치환 + 블로그에 링크 없으면 "찾아오는 길·예약: {URL}" 한 줄 추가.
  - **X 제외**(외부 링크 = 도달 50~90% 감소 정책 유지), 영상 제외.
- 인쇄물 QR(P4): `printable.generate(with_qr=True)` — `src=qr&content=print_{타입}` QR을
  전단/메뉴판/POP 하단에 삽입(옵션, 기본 켬). 목적지 없으면 삽입 안 함.

## 대시보드 (P2)

- 내 콘텐츠: 세트별 "이 콘텐츠로 온 손님 N명" 뱃지(90일, 피스 합산).
- 리포트 탭 "콘텐츠 성과 · 링크 클릭 실측": TOP 3(제목·채널·클릭수), 채널별 유입 30일
  (온라인 채널 vs 매장 QR(오프라인) 구분), 최근 7일 추이 막대. 보라 강조·숫자 크게.

## 성과→학습·브리핑 (P3)

- 생성 주입: 30일 클릭 1위 콘텐츠(≥3회 — 우연 클릭 왜곡 방지)의 키워드·앵글을
  `[성과 학습 — 클릭 실측]`으로 다음 생성 프롬프트에 강화(복제 금지 지시 포함).
- 리포트 코칭: "후기형·'○○' 글이 잘 됐어요 — 다음 글도 이 방향" + 원클릭 생성 링크.
- 아침 브리핑: 어제 클릭 있으면 "어제 콘텐츠가 손님 N명을 데려왔어요(추적링크 클릭 기준)" 동기부여.
- 저녁 피드백: 기존 `clicks_today` 실측 그대로(오늘 실행한 가게만 발송 — 비스팸).

## 한계·후속

- 클릭 ≠ 방문 완료 ≠ 구매. "손님 N명" 표현은 뱃지/동기부여 문구에서만 쓰고 기준(클릭)을 병기.
- 봇/중복 클릭 필터 없음(v1) — UA 기록은 남기므로 필요 시 후처리 가능.
- 네이버 애널리틱스/인스타 인사이트 연동은 공식 API 열리는 범위에서 후속 과제.
